# 字数压缩流程

## 平台限制

种草笔记正文限制 ≤1000 字（含 emoji 和空格）。超过无法发布。

## 压缩流程（仅在字数 >1000 时触发）

### 步骤 1：字数检测

生成完整正文后，统计字数（含 emoji、空格、换行）。若 ≤1000 字，跳过压缩。

### 步骤 2：信息点标记

扫描正文，标记所有核心信息点（具体数据、可操作建议、独特观点）。这些不可删除。

### 步骤 3：三级压缩（按优先级执行）

**优先级 1 — 保留层（不可删）**：
- 核心信息点和数据
- emoji（视觉锚点，删除后影响可读性）
- 开头钩子和互动收尾（结构必需）

**优先级 2 — 精简层（可压缩）**：
- 冗余的修饰语和形容词（"真的非常特别的好" → "很好"）
- 重复表达同一意思的句子（合并为一）
- 过渡句（"那么接下来我们看看" → 直接进入内容）

**优先级 3 — 裁剪层（可删除）**：
- 装饰性铺垫（"说实话"、"不知道大家有没有这种感觉"）
- 多余的互动引导（保留一个即可）
- 可合并的短段落（两个说类似事情的段落 → 一个）

### 步骤 4：合并与压缩

按优先级 2→3 逐步精简，每轮后重新计数。目标：≤950 字（留 50 字安全余量）。

### 步骤 5：验证

压缩后检查：
- [ ] 字数 ≤1000
- [ ] 所有核心信息点保留
- [ ] emoji 保留完整
- [ ] 三段式结构完整（钩子→内容→CTA）
- [ ] 语气一致，无明显删减痕迹
