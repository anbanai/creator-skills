# 爆款要素分析参考

爆款笔记不是运气，是可拆解的工程。以下五大要素决定笔记的互动表现。

---

## 五大爆款要素

### 1. 选题角度（权重 35%）

选题决定了天花板。好选题即使内容一般，互动也不会太差。

- **切入时机**：节日/季节/热点事件的窗口期（提前 3-5 天布局，当天发布最佳）
- **角度差异化**：同主题下的独特切入——"别人教你怎么做，你教别人怎么避坑"
- **受众精准度**：命中高活跃人群（年轻女性 > 30+ 女性 > 职场人士，活跃度递减）
- **情绪共鸣度**：选题本身是否触发强烈情绪（焦虑/好奇/惊喜/愤怒）

### 2. 标题吸引力（权重 25%）

标题是用户和内容的唯一中介。标题失败，内容再好也没人看到。

- 情绪强度（详见 [title-psychology.md](title-psychology.md)）
- 关键词密度（核心词出现在前 10 字）
- 点击冲动触发（信息缺口、数字、疑问、反差）
- 长度优化（12-18 字最佳）

### 3. 内容价值密度（权重 20%）

用户花时间看完后，必须觉得"值得"。

- **可操作性**：读者能否看完立刻行动（"去做X" > "了解X"）
- **信息独特性**：是否提供新知识/新视角（"原来如此" > "我知道"）
- **收藏动机**：是否值得保存备查（清单/步骤/对比表是天然收藏诱饵）
- **信息密度**：每段至少一个具体信息点（数字/场景/方法/案例）

### 4. 视觉吸引力（权重 10%）

图片是种草笔记的核心载体，文字是辅助。

- 封面点击率（0.5 秒决策，详见 visual-design skill）
- 内容图完读率（信息密度适中，视觉一致性）
- 视觉独特性（区别于同主题其他笔记的图片风格）

### 5. 互动设计（权重 10%）

互动是算法推荐的核心信号。没有互动设计的内容，算法不会持续推流。

- **评论门槛**：低参与门槛的互动设计（选择题 > 开放题 > 无引导）
- **分享动机**：是否有转发给他人的理由（"转给闺蜜看看"）
- **关注理由**：是否有持续关注的价值（暗示系列内容或更多同类）

## 爆款 vs 普通笔记对比

| 维度 | 爆款笔记 | 普通笔记 |
|---|---|---|
| 标题 | 情绪词 + 数字 + 悬念 | 纯信息描述 |
| 开头 | 场景/故事/痛点代入 | "今天来分享..." |
| 内容 | 每段一个可操作点 | 泛泛而谈 |
| 结尾 | 具体互动问题 | "欢迎关注" |
| 图片 | 信息图 + 手绘批注 | 纯文字截图 |
| 标签 | 核心词 + 垂直 + 长尾 | 只用大话题 |
| 价值感 | "这篇我得收藏" | "看完就忘了" |

## 爆款公式速查

1. **痛点 + 解决方案**：痛点描述 → 具体方法 → 即时效果（适用：教程、攻略）
2. **反差 + 揭秘**：打破认知 → 真相分析 → 实操建议（适用：知识分享、行业揭秘）
3. **合集 + 测评**：选品逻辑 → 对比维度 → 推荐结论（适用：好物推荐、产品对比）
4. **经历 + 复盘**：真实经历 → 踩坑总结 → 经验提炼（适用：个人成长、职业分享）

## 写作自检清单

- [ ] 标题含情绪词或数字（12-18 字）
- [ ] 开头有场景代入或痛点共鸣（非"今天来分享"）
- [ ] 每段至少一个具体信息点（数字/场景/方法）
- [ ] 结尾有明确互动引导（非"欢迎关注"）
- [ ] 至少一个"收藏理由"（可保存的实用信息）
- [ ] 内容有信息缺口（让用户产生好奇）
- [ ] emoji 作为视觉锚点，无 Markdown 残留
- [ ] 字数 ≤1000 字

## A/B 候选交付（可选，让真实数据反哺）

标题和封面是点击率（CTR）最大的两个变量，但**谁也说不准哪个更好**——只有真实数据知道。所以当一篇笔记值得冲爆款时，与其押一个，不如留一个 A/B 出口：

- **标题双候选**：§1 已生成 3 个候选并自动评分选 Top1。当两个候选评分接近（差距 ≤1 分）时，可在 `content.md` 顶部备注一个"备选标题 B"，供发布方择优测试。
- **封面双候选**：高潜力选题（CES 评分高、或用户明确要冲爆款）时，可额外生成 1 张备选封面（不同封面类型，如笔记感 vs 大字报），交付时一并给出。
- **发布时机**：黄金时段（7-9 / 12-14 / 18-20 / 21-23，见 algorithm-optimization.md）首发；若首版 2 小时互动低于预期，换备选标题/封面重发或作为下一篇参考。

> 这条**不强制**——自动流水线默认交付单版最优即可，避免拖慢产出。仅当用户明确要冲爆款、或候选间难分伯仲时启用。它的价值在于：把"猜哪个标题好"变成"用数据选哪个标题好"，长期提升账号的爆款命中率。
