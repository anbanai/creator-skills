# 种草笔记内容写作知识库

## 案例库

遇到场景分支、产物格式或质量边界不确定时，先读 [references/examples.md](references/examples.md)。

## 图片比例固定规则

本 Skill 只要涉及生成、选择、裁切、校验或引用图片，必须按以下优先级决定画面比例：

1. 用户/任务明确指定的 `image_ratio`、`size` 或平台规格优先。
2. 项目/频道默认比例次之。
3. 业务默认比例只作兜底：微信文章封面/正文图默认 `16:9`；Seednote/XLS/移动信息流默认 `3:4`；电商、广告投放、视频封面按具体平台素材位要求执行。
4. 不得从模型路由、供应商默认 `size` 或模型能力反推业务比例；模型只决定能力和成本，比例属于创作场景约束。


## 0. 用户输入锁定规则

当用户 prompt 明确提供以下字段时，默认视为用户指定内容，不得擅自替换主题、标题或标签：

- `封面标题`：作为最终发布标题和封面主文案，除非用户明确要求优化标题
- `笔记正文`：必须保留核心主张、茶类/产品/场景、关键数字和表达顺序，只做平台化润色
- `话题标签`：必须完整保留用户给出的标签，可补充 1-3 个相关标签，但不得删除原标签

如果生成候选标题评分高于用户指定标题，也只能记录为"可选优化标题"，不能覆盖用户指定标题。所有后续图片规划必须围绕用户指定字段，不得扩展到不相关季节、品类或下一期主题。

## 1. 标题规范

**基本要求**：≤20 字，关键词前置，情绪词优先于纯信息词。

**常用句式速查**：
- 利益驱动：`带你学会___`、`让你掌握___`、`马上 get ___`
- 对号入座：`小个子/初次备孕/大学生___`（直接写出目标用户身份标签）
- 痛点代入：`为什么___？`、`如何做___？`、`___怎么办？`
- 数据具象：`7天搞定___`、`45款___测评`、`6款___必备`
- 权威背书：`TED 推荐`、`人民日报｜___`、`国家出品___`
- 危机警示：`千万别___，否则___`、`想避免___？立即停止___`

**标题评分公式**（内部生成 3 个候选，自动选 Top 1）：
```
title_score = 情绪强度(0-3) + 关键词密度(0-1) + 字数优化(0-1) + 句式加分(0-1)
情绪强度: 含强情绪词→3, 中情绪→2, 弱情绪→1, 无→0
字数优化: 12-18字→1, 否则→0
句式加分: 疑问/否定/数字句式→1, 否则→0
```

详细公式与案例：[title-psychology.md](references/title-psychology.md)
写作技巧：[title-techniques.md](references/title-techniques.md)

## 2. 正文结构（三段式）

1. **开头钩子 / 黄金前两行**（1-2 句）：前两行是用户决定"继续看还是划走"的分水岭，也是 2026 算法读取**停留 / 完读 / 深度互动信号**的关键窗口——必须命中「痛点 / 共情 / 反差 / 数字 / 悬念」之一。用具体场景或感官细节代入（"上周去咖啡馆，无意间发现…"），**禁止**"今天来分享/本文将介绍"式抽象开场。开场钩子方式见 §3。
2. **核心内容**（2-3 段）：观点 → 证据 → 补充/反方视角
3. **互动收尾**（1 句）：开放式思考问题，不引导具体互动操作。禁止包含"评论区"、"点赞收藏"等引导语，禁止以"👇"指向评论区。正确示例："你们平时是怎么做的？"而非"你们怎么看？评论区告诉我"

**段落格式**：
- 短段落，每段 2-4 行，避免大段文字堆砌
- 适当使用 emoji 增加趣味性（尤其针对年轻女性受众）
- 语气因受众而异：年轻女性→幽默温柔感性；职场内容→专业严谨

## 2.5 正文格式（关键 Gotcha）

**平台陷阱**：种草笔记不渲染 Markdown。`**粗体**` 在用户手机屏幕上就是 `**粗体**` 四个字符，`---` 就是三个横杠。这是平台硬限制，不是风格偏好。

发布到平台的正文必须是**纯文本 + emoji + 符号**。内部文档（topic-analysis.md、source-analysis.md、image-plan.md）不发布，可用 Markdown。

**格式设计三原则**：

1. **丰富的 emoji 是视觉锚点** — 用户刷信息流时，emoji 先于文字被注意到。把 emoji 放在段落开头或关键词旁，引导注意力
2. **空行是唯一可靠分隔** — 没有 `<hr>`，没有 `---`，换行两次就是分段，这是唯一有效的视觉分隔
3. **格式跟随内容调性** — 知识干货用序号 emoji（逻辑感强）；情感故事用氛围 emoji（松散排列）；测评对比用表格式 emoji 对比（直观）

**三种内容类型范例**：

知识类（咖啡馆选址攻略）：
```
选址是开店成败的 ✨第一关✨ 📍

❀ 三个核心指标 ❀

1️⃣ 人流量 — 早高峰过人数 > 500 是 ✨基准线✨
2️⃣ 租金比 — 月租不超过预期月营业额的 ✨15%✨
3️⃣ 竞争密度 — 500m 内同类型咖啡馆 ✨≤2 家✨

💡 选址前先蹲点，连续 3 天统计早中晚客流
```

情感/生活类（旅行记录）：
```
第一次一个人去京都

到的那天下着小雨，祇园的石板路全是水光 🌧️

没有行程表，没有攻略，就这样走着

路过一家小店，老板不会说中文
我不会说日语
但他给我倒了一杯热茶 🍵

有些瞬间，不需要语言
```

测评/对比类（手机摄影 App 测评）：
```
用了半年终于敢说哪个 ✨最好使✨ 📱

✧ 摄影App实测对比 ✧

Snapseed  ✅ 免费 / 曲线精准 / 学习成本高
VSCO      ✅ 滤镜漂亮 / 有社区 / 订阅才能用高级功能
Lightroom ✅ 专业全面 / 适合 raw 格式 / 稍重

日常随手拍 → ✨Snapseed✨
追求氛围感 → ✨VSCO✨
认真摄影 → ✨Lightroom✨

别三个都装，根据自己定位 ✨选一个用熟✨
```

### Markdown → Emoji 替换

生成内容时直接使用 emoji + 纯文本，不使用 Markdown 语法。详细映射表（基础映射、标题/分段符号映射、粗体→emoji 上下文选择、包裹规则、调性→emoji 集匹配）见 [emoji-mapping.md](references/emoji-mapping.md)。

核心规则速记：粗体用 emoji 包裹（✨重点✨），列表用序号 emoji（1️⃣2️⃣3️⃣），标题用装饰符号（❀ 标题 ❀），空行是唯一分隔符。

## 2.6 字数压缩（强制步骤）

**平台限制**：种草笔记正文限制 ≤1000 字（含 emoji 和空格）。超过无法发布。

完整压缩流程（三级压缩：保留层→精简层→裁剪层）见 [word-compression.md](references/word-compression.md)。

速记：标记核心信息点（不可删）→ 压缩冗余修饰和过渡句 → 删除装饰性铺垫。目标 ≤950 字。

## 2.7 内置去 AI 味

正文（含标题）定稿后，本 Skill 直接做轻量去 AI 改写，再接合规检查。不要再调用 `humanizer` Skill；Seednote 只需要下面这些业务相关规则，额外加载通用 33 类规则会浪费上下文。重点针对：

- 空洞升华 / 过度总结（"让我们拥抱美好的一天"、"开启全新体验"）
- 三段式套话与"不仅…而且"式否定排比
- AI 高频词（"赋能 / 打造 / 彰显 / 至关重要 / 深入探讨"等书面腔）
- 论文腔开头与"综上所述"式收尾

**改写而非删除**——保留种草笔记的人称代入（"我/集美/姐妹"）、情绪节奏、具体场景细节与原生口语表达（"针不戳"、"绝了"）。这是自动流水线步骤：不得调用 `AskUserQuestion`；没有写作样本时直接按账号画像和当前稿件语气改写。本步骤不调用任何 MCP 工具、不计费、无强度档位，且不得在改写中引入新的违禁词、虚假承诺或诱导互动表达。

只在多个迹象成簇出现时改写，不把单个 emoji、口语词、短句或自然情绪当作 AI 痕迹。改写应具体、克制且保留原有信息点。

> 注意：去 AI 改写可能改变字数，改写后须复核是否仍 ≤1000 字（超出则回到 §2.6 压缩）。

## 3. 写作语气与沉浸感

- **对话式第一人称**：像跟朋友聊天，用"我"和"你/集美/姐妹"，拉近距离
- **沉浸式场景开头**：感官细节或微故事开场（"上周去咖啡馆，无意间发现..."），不要抽象陈述（禁止"本文将介绍..."）
- **平台原生表达**：自然使用种草笔记惯用语（"针不戳"、"绝了"、"已上车"），不堆砌，不刻意

**反面模式（禁止）**：
- 论文腔："本文旨在探讨..."、"综上所述..."
- 第三人称客观叙述："用户可以通过..."
- "本文将介绍"式开头
- 过度专业术语堆砌（除非垂直专业赛道）

## 4. 话题标签

- **数量**：5-8 个（少则曝光不足，多则稀释权重）
- **组合**：核心大话题（1-2 个）+ 垂直话题（2-3 个）+ 长尾精准话题（2-3 个）
- **格式**：`#话题名` 放在正文末尾
- **选取原则**：优先平台热门话题；结合当下季节/热点；直击目标用户痛点

## 6. 受众定位

- **卖点差异化**：先找出与竞品相比与众不同之处，在文案中放大突出
- **人群画像**：确定受众后才能确定文案书写方式（语气、词汇、emoji 用量）
- **语言风格适配**：
  - 年轻女性：幽默感 + emoji + 温柔感性语气 + 网络流行词（"集美"、"针不戳"）
  - 30+ 消费女性：生活化 + 有代入感的品质叙述
  - 职场人士：专业严谨 + 数据/案例支撑
- **账号定位一致性**：人设需贯穿每一篇笔记，避免风格差异过大

## 8. 风格一致性原则

- **标题/正文/封面三者风格统一**：风格差异会让用户质疑笔记专业度
- **账号调性一致**：单个账号内文案风格不宜差异过大，否则用户对账号真实性产生怀疑
- **萌宠类账号**：多用 emoji、可爱词汇；**职场教育类**：严肃严谨、专业用词
- 标题与封面文字的字体风格需基于账号内容定（不要萌宠用厚重字体、职场用可爱字体）

## 9. 爆款改写

复刻模式下，先由 `seednote-viral-analysis` skill 生成 `$DIR/viral-template.json`，本 skill 只读取该模板并输出高贴合主题的可发布新笔记。不要重新拆解源笔记，不要调用 `save_template`。

支持 3 种改写模式：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| `style-only`（默认）| 保留原主题与互动机制，封面只参考风格/色调/信息层级，不复用具体元素 | 大多数情况 |
| `medium` | 保留主题方向，适度借鉴结构，内容重新设计 | 需要兼顾原帖热度和原创度 |
| `tight` | 保留同主题、同互动机制、同内容结构，仅替换具体措辞和案例 | 测款阶段快速复制爆款结构 |

### 9.1 模板输入

读取 `$DIR/viral-template.json`，至少包含：

- `title_template`
- `cover_template`
- `body_template`
- `interaction_template`
- `tag_template`
- `audience_insight`
- `viral_mechanism`
- `rewrite_constraints`
- `do_not_copy`
- `recommended_clone_depth`
- `confidence`

若模板文件缺失或 JSON 无效，停止复刻写作并要求先执行 `seednote-viral-analysis`。不要凭空重建模板。

### 9.2 改写规则

**tight 模式**（高一致性）：
- 保留：同主题、同互动机制（如"评论区打确认"）、同内容结构
- 替换：具体措辞、案例细节、账号人设口吻
- 禁止：逐句照抄、原图二次使用、原作者专属信息、`do_not_copy` 列出的元素

**style-only 模式**（默认）：
- 封面：仅参考风格、色调、信息分层与竖版构图
- 明确禁止复用：人物姿势、图标组合、文本框形状与位置
- 使用"保留主题 + 重设计元素"策略

模式选择规则：

- 默认使用 `viral-template.json.recommended_clone_depth`
- 若用户指定更高强度，但模板 `confidence=low` 或 `do_not_copy` 风险多，自动降级并记录原因
- `style-only` 只迁移机制和风格方向；`medium` 可迁移结构但重写案例；`tight` 仅在低风险、强通用结构下使用

### 9.3 标题生成

基于 `title_template` 和当前账号画像生成 3 个候选，使用本 SKILL 第 1 节标题评分公式自动选 Top 1（≤20 字）。标题不得复制源笔记标题，不得只替换一两个词。

### 9.4 内容输出格式

1. **标题**：自动评分选定的最优标题（≤20 字）
2. **正文**：1 版可直接发布，遵循本 SKILL 正文结构（不含话题标签）
3. **话题**：5-8 个（单独列出）
3. **字数限制**：≤1000 字。生成后若超字数，按第 2.6 节压缩流程处理

### 9.5 合规检查

加载第7节违禁词库，扫描标题与正文：

- **高风险词**：删除所在句子或重写段落，超过 3 处触发全文重写
- **中风险词**：自动替换为合规近义词
- **低风险词**：自动替换或删除

**诱导互动专项扫描**（在上述违禁词扫描之后执行）：

扫描以下模式，发现后自动删除或改写：

| 违规模式 | 处理方式 |
|---------|---------|
| 含"评论区"字样 | 删除该句或改写为不含"评论区"的开放式问题 |
| "点赞收藏"四字组合 | 删除或拆分 |
| "收藏" + "不迷路/防走丢" | 删除该句 |
| "关注" + "送/领/回粉/抽奖" | 删除该句 |
| "双击屏幕"、"长按点赞" | 删除该句 |
| "👇" 出现在正文末尾（指向评论区） | 删除该符号 |

合规替代原则：提出开放式问题但不指定互动渠道（"你们平时怎么做的？"），或以内容本身的实用性（清单/步骤/对比表）驱动自然收藏。

**AI 生成合成内容标识（2026 强制执行，新增）**：本管线用 AI 写作 + AI 出图。国家《人工智能生成合成内容标识办法》（2025-03 发布、**2025-09-01 施行**，2026 持续收紧执行）+ 小红书薯管家公告要求 **AI 生成合成内容须主动标识**——未标识→限制分发/限流/下架，恶意隐瞒→封号；平台会对未声明的疑似 AI 内容自动打"疑似使用 AI 生成技术"标。处理原则：
