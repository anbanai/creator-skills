# 参考封面分析模板

本文档定义 `analyze_image` 调用时的 prompt 模板，用于在 Phase 1 步骤 3 拆解参考封面。

## 调用语法

```
analyze_image(
  project_id="$PROJECT_ID",
  file_path="$REF_SERVER_PATH",
  prompt=<下方完整模板>
)
```

## 8 维度分析 prompt 模板

直接复制粘贴这段作为 `prompt` 参数：

```
分析这张短视频封面图，从以下 8 个维度输出结构化描述。每个维度提供具体、可迁移到新封面的视觉逻辑，而不是单纯描述"是什么"。

## 1. 画面比例
- 实际比例（如 9:16 / 16:9 / 1:1 / 3:4）
- 横竖方向
- 主体在画面中的占比（如"人物占画面 60%"）

## 2. 标题位置与层级
- 标题整体位置（顶部 / 中部 / 底部 / 左 / 右 / 居中）
- 标题分几行
- 主标题与副标题的字号差异（如"主标题字号是副标题的 3 倍"）
- 标题与主体的空间关系（叠加 / 分离 / 环绕）

## 3. 人物/主体位置
- 是否有人像；有则描述：朝向（正面 / 侧面 / 背影）、表情倾向、服装色调
- 主体（人物或核心物体）在画面中的具体位置（中心偏左 / 右上 / 占据整个画面）
- 主体与标题的视觉权重对比（谁更突出）

## 4. 背景氛围
- 背景类型（纯色 / 渐变 / 实景 / 抽象图形 / 模糊处理）
- 整体色调（暖色 / 冷色 / 中性 / 高对比黑白）
- 复杂度（极简 / 中等 / 复杂纹理）
- 与主体的关系（衬托 / 对比 / 融合）

## 5. 主色与强调色
- 主色：识别 1 个主导色，用语义名描述（如"深夜蓝"而非"#0A1A2F"）
- 强调色：识别 1-2 个对比色，用实物类比描述（如"蜂蜜般的暖黄"）
- 主色与强调色的关系（互补 / 类似 / 三色组合）
- 色彩饱和度倾向（高饱和 / 中等 / 低饱和）

## 6. 字体气质
- 字体类型判断（黑体 / 宋体 / 圆体 / 手写体 / 装饰体）
- 字重（细 / 常规 / 粗 / 特粗）
- 整体气质（严肃专业 / 活泼俏皮 / 高端商务 / 接地气 / 文艺）
- 是否有特效（描边 / 阴影 / 渐变填充 / 立体感）

## 7. 视觉重点
- 第一眼看到的是什么（标题 / 人脸 / 物体 / 色块）
- 视觉动线（眼睛从哪里开始，按什么顺序扫过画面）
- 哪些元素被刻意强化（放大 / 高亮 / 居中 / 加特效）

## 8. 构图技巧
- 用了什么经典构图法（三分法 / 黄金分割 / 对称 / 引导线 / 框中框）
- 留白处理（极满 / 适度 / 大面积留白）
- 元素之间的层级关系（前后景 / 主体与背景的虚实对比）

输出格式：每个维度用 ## 标题分节，每节 2-4 句话具体描述。不要笼统说"画面好看"，要给出"新封面可以借鉴的具体视觉逻辑"。
```

## 输出落地

`analyze_image` 返回的文本直接作为 `$DIR/reference-analysis.md` 的正文，文件开头加：

```markdown
# Reference Cover Analysis

## Source

- file: $DIR/input-manifest.md 中的 reference_cover
- analyzed_at: <时间戳>
- ref_server_path: $REF_SERVER_PATH

---

<analyze_image 返回内容>
```

## 后续使用

`reference-analysis.md` 是后续 Phase 2（迁移到新内容）和 Phase 3（构建 prompt）的依据。Phase 4 审计时也要回头比对——判断生成结果是否真的"学到了视觉逻辑"，而不是"画了一张类似的图"。
