# 电商视觉设计原则（高级设计师方法论）

> 本文件是「达到高级设计师水平」的方法论底座。原则服务于**点击与下单转化**——美观是手段，不是目的。每条原则都解释「为什么」，让生成 prompt 能讲清意图而非堆形容词。

## 一、字体（中文电商字体系统）

### 选型：按调性匹配，全店一致

- **阿里巴巴普惠体 / 思源黑体（Source Han Sans）**：通用安全，现代干净，适合 3C/食品/日用百货——可读性最强，移动端不糊。
- **思源宋体（Source Han Serif）/ 站酷文艺**：高级感、文化感，适合美妆/茶/高端食品/礼盒——衬线传递品质与故事。
- **站酷快乐体 / 手写体**：亲和、活泼，适合零食/母婴/宠物——传递情绪与趣味。
- **原则**：**一个任务只用 1-2 个字族**（标题字族 + 正文字族），全图一致；混用超过 2 个字族 = 业余感。数字/价格可用更粗的等宽或加粗黑体强调。

### 字号层级（移动端首屏可读是硬指标）

电商图以移动端浏览为主，字号按「视觉权重」分 4 级，比例约 **标题 : 副标 : 正文 : 角标 ≈ 3 : 2 : 1.4 : 1**：

- **主标题/主卖点**：图宽的 ~12-15%，移动端等效 ≥48px（让买家扫一眼就读到）。
- **副标题/辅卖点**：~8-10%，≥32px。
- **正文/参数/说明**：≥24px 等效（移动端可读下限）。
- **角标/价格/促销**：小但醒目，靠颜色/底色对比突出，不必靠字号。
- **字间距/行高**：中文标题略收紧（letter-spacing -2~-4px 等效）更紧凑有力；正文行高 1.4-1.6 倍字号保证多行可读；标题单行优先。

### 写进 prompt

明确「标题用思源黑体加粗、字号约为图宽 14%、字距收紧；正文思源黑体常规、≥24px；全图仅此一字族」——比「大字」「清晰」有用得多。

## 二、色彩（从产品主色衍生配色系统）

### 从产品主色 HEX 衍生（保证品牌一致 + 协调）

拿到产品档案的**主色 HEX** 后，按和谐配色法衍生 3-5 色调色板，写入 `$STYLE`：

- **同类色/单色（monochromatic）**：主色 + 同色相不同明度（主色、主色+白、主色+黑）。最安全，高级感强，适合极简/高端。
- **邻近色（analogous）**：色相环相邻 2-3 色（如蓝+蓝绿+绿）。和谐自然，适合生活方式/场景。
- **互补色（complementary）**：主色 + 对比色（如橙+蓝）。**最强视觉冲击**，适合促销/钩子/价格强调，但仅用 1 个互补色作点缀，避免打架。
- **三角/分裂互补**：丰富但不乱，适合多品类 banner。

### 色彩心理与电商转化

- **信任/专业**：蓝、深蓝、灰（3C/金融/服务）。
- **食欲/促销/紧迫**：红、橙、黄（食品/促销/限时）。
- **高级/纯净**：黑、白、金、低饱和中性（美妆/高端/极简）。
- **自然/健康**：绿、棕、米（生鲜/有机/个护）。
- **背景**：默认白/浅灰（契合白底图规范）；深色产品用浅背景提反差，浅色产品用深背景或彩色背景增层次。

### 对比度（可读性底线）

文字与背景对比度要够（参考 WCAG AA，对比比 ≥4.5:1）。**深字配浅底、浅字配深底/深色蒙层**。卖点文字若压在产品/场景上，须加底色块/半透明蒙层/描边保证可读——避免「好看但读不清」。

## 三、构图（让买家看到你想让他看的）

### 三分法 / 九宫格 / 黄金分割

- 把画面横竖三等分得九宫格，**主体与视觉焦点落在四条线的交点（黄金分割点）上**，比正中更生动平衡。
- 主图①产品主体常占图 **50-65%**，偏左或偏右三分之一位，留出另一侧给文案/留白——这是「主体最大但呼吸感足」的原因。比例按品类调：服饰模特可更大，3C 产品适中，食品诱人特写可铺满。

### 视线引导（F 型 / Z 型阅读路径）

- **F 型**：文字密集的详情页/参数区，买家视线从左上→右下横向扫，重要信息沿左侧纵轴 + 顶部横轴排。
- **Z 型**：主图/海报，视线左上→右上→左下→右下对角流动——把 logo/钩子放左上，产品放中，价格/CTA 放右下。
- **视觉锚点**：用一个最强元素（最大/最亮/最高对比）抓住第一眼，再用大小/明暗/颜色引导到次级信息。

### 留白（呼吸感 = 高级感）

主体四周留白，文字不压在产品关键部位（logo/正面/质感区）。留白越足越显高级；堆满 = 牛皮癣 = 廉价 + 违规风险。

## 四、信息层级实现（主体最大 → 卖点次之 → 钩子醒目）

不是写「信息层级清晰」，而是用 5 个变量实现：

| 变量 | 主角（产品/主卖点） | 配角（次卖点/参数） | 钩子（价格/促销） |
|---|---|---|---|
| **大小 size** | 最大 | 中 | 小 |
| **字重 weight** | 最粗 | 常规 | 加粗 |
| **位置 position** | 视觉中心/黄金点 | 环绕/下方 | 角标/边缘醒目位 |
| **颜色 color** | 主色/高对比 | 中性灰/辅色 | 互补色/红黄警示 |
| **对比 contrast** | 最强 | 中 | 靠底色块突出 |

- **一个画面一个主角**：第一眼必须落到你想强调的（主图①= 产品+第一卖点）。多个元素同等大小 = 没有层级 = 买家不知道先看什么。
- **何时打破层级**：为强调促销/限时，可让「钩子」虽小但用最高对比色（红底白字角标）跳出来——小而亮 > 大而平。

## 五、移动端适配（首屏即转化）

- **首屏安全区**：核心卖点与主体落在中心 80% 区域，**避开平台水印位（右下价格标、底部导航、顶部搜索条占位）**——淘宝/京东/抖音/微信的价格标常在右下角，主图①关键信息不要压在那。
- **最小可读字号**：正文 ≥24px 等效，标题更大；手机端缩略图（淘宝搜索 60×60、推荐位）下，主图①的第一卖点要还能辨认——所以主图①文字宁少宁大。
- **触控热区与密度**：信息点之间留间隙，不挤成一团；详情页单节聚焦 1 个卖点 + 2-4 支撑点，宁分节勿堆砌。
- **竖屏优先**：抖音/小红书/微信详情多为竖屏，3:4 / 9:16 构图把「钩子→主体→卖点」自上而下排进首屏。

## 六、把原则翻译成 prompt（关键）

生成 prompt 时，把上面原则转成**具体可执行的描述**，而非模糊形容词：

- ❌「高级感、美观、信息清晰」（模型无法执行）
- ✅「产品主体居中偏左、占图 60%；标题思源黑体加粗置于右上黄金分割点、字号约图宽 14%、深灰；价格角标红底白字置于右下；背景纯白 + 浅灰投影；四周留白；整体仅思源黑体一字族；标题/价格/正文三级字号比约 3:1.5:1」

这样模型才有可执行的设计意图。设计原则的价值，就是让每张图的 prompt 都能讲清「主角是谁、配角是谁、视线怎么走、字号字色怎么定」。
