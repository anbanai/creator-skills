## 内容页设计规范

**内容密度**：
- 文字占比：20-35%
- 每页字数：≤60 字
- 每页信息点：2-4 个，每点 ≤15 字
- 留白：35-45%

**6 种布局模式**（按内容类型选）：
1. **编号清单** — 步骤、要素列举
2. **对比双栏** — Before/After，优劣对比
3. **步骤流程** — 有先后顺序的操作
4. **标注图解** — 图片+手绘说明
5. **数据卡片** — 数字/数据展示
6. **Q&A 对话** — 问答式知识

**视觉规范**：
- 字体：圆角/手写，避免黑体/等线
- 高亮方式：色块或荧光线标记，勿加粗
- 背景：真实照片场景（每张不同场景）
- 批注：手绘风格，与封面保持一致
- 色调：与封面统一
- 中文主题只允许出现简体中文；禁止英文翻译、拼音、乱码、伪词和自动补充外文标签
- 当页文字必须来自 image-plan 的"必须出现文字"，不得自行创造新术语或改写关键数字

**视觉多样性**（多张内容图时，2~3 张均适用）：
- 3+ 种不同实景背景
- 构图角度变化（正视/俯视/局部特写）
- 焦点位置错落（左/中/右交替）
- 批注样式轮换（不完全重复同一种）

**禁止**：
- 3D 渲染/纯色大背景
- PPT 式对称布局
- 科学海报风格
- 无结构文字墙
- 连续 3 张相同视觉元素
- 手机框/设备框
- 与当页主题无关的茶类、季节或产品
- 误导性冲泡动作，例如把"10秒出汤"写成"焖泡10秒"

## 图片内容规划（image-plan.md）

基于选题方向、最终标题和目标受众，在生成图片前先独立规划每张图的具体内容，写入 `$DIR/image-plan.md`。

**图片总数**：1~5 张，由 `seednote_image_mode` 决定：`cover_only` 仅封面，`cover_content` 为封面 + 内容图，`cover_tail` 为封面 + 尾图，`full` 为封面 + 内容图 + 尾图。封面始终生成；内容图仅在 `cover_content` / `full` 模式生成，张数按信息点分组自适应（最多 3 张）；尾图仅在 `cover_tail` / `full` 模式生成，不含尾图的模式 `image-plan.md` 不含 `## tail` 节。所有图片逐张调用 `generate_image` 生成。

封面、内容图和尾图均不预设是否使用参考素材。每页根据 `image-plan.md` 独立选择 0、1 或多张原图；没有相关参考时使用纯文生图。项目级品牌参考图仍可作为旧数据来源，但不得覆盖本次输入附件中更具体、更新的产品事实。

**格式模板**：

```markdown
# 图片内容规划

## 总体策略

- 主题方向: xxx
- 目标受众: xxx
- 图片内容定位: xxx
- 计划图片数量: N 张

---

## cover 封面

- 钩子: （≤10字）
- 辅助信息: （≤15字）
- 必须出现文字: （1-2 行简体中文主文案）
- 视觉主体: （能直接看出主题的实物/场景）
- 禁止元素: （英文、伪词、错别字、无关品类、医疗功效承诺）
- 验收标准: （0.5 秒内能读懂主题）

---

## image_01 [内容] 主题：xxx

- 信息点1: （8-15字）
- 信息点2: （8-15字）
- 推荐布局: 编号清单 / 对比双栏 / 步骤流程 / 标注图解
- 必须出现文字: （2-4 条简体中文短句）
- 视觉主体: （当页知识对应的实物/过程/对比对象）
- 禁止元素: （英文、伪词、无关主题、误导性参数）
- 验收标准: （文字准确 + 主体准确 + 与 image-plan 一致）

---

## tail [尾部] 类型：{follow|comment|traffic}（**仅当 `seednote_image_mode` 包含尾图时添加本节；不含尾图则整节省略**）

匹配依据：{为什么选这个类型}
- 内容点: （见 tail.md 规范，根据类型填充）
```

## Prompt 模板（v2 - 中文文字渲染优化）

**内容页**：
```
{page_content}

页面类型：内容图（{page_type}），种草笔记信息流内容
风格延续：{style}（与封面保持一致）

平台规范：
- 3:4 竖版构图
- 真实照片背景（与其他内容图不同场景）
- 圆角/手写字体，荧光色块高亮核心词
- 手绘批注风格
- 文字占比 20-35%，留白 35-45%

【文字要求 — 重要，违反则视为生成失败】
必须显示的中文短句（按列表顺序，每条独立成行，禁止合并/拆分/修改任何字符、增删字、改标点）：
1. 「{信息点1}」
2. 「{信息点2}」
3. 「{信息点3}」

每条短句使用圆角手写体，字号占图宽 6-9%，深色字配浅色背景或白色字配深色背景。
按内容图布局模式（编号清单/对比双栏/步骤流程/标注图解/数据卡片/Q&A对话）排版，
每条短句对应一个独立信息块，留白充足。

【禁止项 — 严格遵守】
- 任何英文字母、拼音
- 乱码、伪词、错别字、字形错误
- 把多条短句合并成一段或拆分原句
- 改写关键数字、茶类名称、冲泡动作（如"10秒出汤"不得写成"焖泡10秒"）
- 在文字区域画装饰物遮挡
- 与当页主题无关的茶类、季节或产品
- 纯色背景、PPT布局、3D渲染、设备框

注：上方列表的「1. 2. 3.」编号是 prompt 元指令（告诉模型共 N 条短句），**不应作为可见数字出现在画面里**；画面上只渲染「」内的文字内容。

内容主题：{topic}
全文概要：{full_outline}
视觉主体（不能省略）：{visual_subject}
验收标准：{acceptance_criteria}
```

**生成方式（逐张调用 generate_image）**：

按 `image-plan.md` 逐张调用 `generate_image`：
- 每张使用对应 section 的信息点填充 `{page_content}` 和 `{page_type}`
- `output_path` 设为 `$DIR/image_01.png`、`$DIR/image_02.png` ... `$DIR/image_03.png`
- `ref_image_paths` 只包含 `image-plan.md` 为当前页选中的原始路径，顺序与 prompt 的参考图编号一致；当前页没有相关参考时省略该字段并使用纯文生图
- 每张使用独立视觉主体、场景、构图和参考子集，禁止与其他内容图机械复用
尾图单独生成见 tail.md（仅当 `seednote_image_mode` 包含尾图时）。

> **关键规则**：封面、内容图和尾图均不预设是否使用参考素材。每页根据 `image-plan.md` 独立选择 0、1 或多张原图；没有相关参考时使用纯文生图。项目级品牌参考图仍可作为旧数据来源，但不得覆盖本次输入附件中更具体、更新的产品事实。 每张内容图必须使用对应信息点构造独立 prompt；共享“风格延续：{style}”文本块只负责调性一致，不能替代对产品事实和参考素材用途的逐页判断。
