# 电商转化文案：卖点提炼 + 主图 + 详情页 + 分享

## 案例库

遇到场景分支、产物格式或质量边界不确定时，先读 [references/examples.md](references/examples.md)。

## 图片比例固定规则

本 Skill 只要涉及生成、选择、裁切、校验或引用图片，必须按以下优先级决定画面比例：

1. 用户/任务明确指定的 `image_ratio`、`size` 或平台规格优先。
2. 项目/频道默认比例次之。
3. 业务默认比例只作兜底：微信文章封面/正文图默认 `16:9`；Seednote/XLS/移动信息流默认 `3:4`；电商、广告投放、视频封面按具体平台素材位要求执行。
4. 不得从模型路由、供应商默认 `size` 或模型能力反推业务比例；模型只决定能力和成本，比例属于创作场景约束。


## 受众与目标

- **受众**：买家/潜在消费者，处于决策路径 **注意 → 兴趣 → 欲望 → 行动（AIDA）**。
- **目标**：主图打赢 **CTR（点击率）**，详情页打赢 **转化率与客单价**，分享图打赢 **站外引流**，整体降低退货（靠信息准确、不夸大）。

这是**商业转化文案**，不是种草的情绪共鸣叙事，也不是品牌 PR 稿——每句话都要推动买家向「点击/下单」移动一步。

## 输入

- `$DIR/product-bible.md`（产品档案，含卖点候选）
- 用户提供的 `selling_points`（可选，优先采纳）
- `target_platform`（影响文案口吻与禁用词）
- 已选模块 `selected_modules`（决定产出哪些文案）

## 步骤 1：卖点提炼（FABE + 多框架）

从产品档案的 `selling_point_candidates` + 用户卖点，提炼 **3-5 个核心卖点**并**按电商转化优先级排序**。

**提炼框架（择一或组合，确保卖点有据）**：
- **FABE**：Feature 特征 → Advantage 优势 → Benefit 利益 → Evidence 证据。每个卖点都要能走完 FABE，避免只讲特征不讲利益。
- **黄金圈**：Why 为什么需要 → How 如何做到 → What 是什么。
- **痛点—解决方案**：买家痛点 → 产品如何解决。
- **价值四象限**：功能价值 / 情感价值 / 社交价值 / 经济价值，覆盖不同买家动机。

**排序原则**（转化优先）：
1. 第一卖点 = 最大差异化 + 最大痛点解决 + 最易在主图①3 秒内传达。
2. 后续按「对决策影响力」降序，证据型卖点（资质/对比/数据）靠后但必须有。

**产出**：`## 核心卖点` 节，每个卖点含：标题（≤10 字）、FABE 展开、证据、目标买家、**建议可视符号**（供 visual-design 把卖点转成图内符号，见下）。

### 卖点 → 视觉符号桥（给 visual-design 用）

每个排序卖点给出**建议可视符号**，让抽象卖点能在图里可视（这是文案与视觉的桥）：

| 卖点类型 | 可视符号示例 |
|---|---|
| 长续航 / 大容量 | 电量条 / 时钟 / 容量柱对比 |
| 轻 / 便携 | 羽毛 / 手持比例 / 与硬币对比 |
| 防水 / 防摔 | 水珠 / IPX 等级标 / 跌落瞬间 |
| 快充 / 速效 | 闪电 / 倒计时 / 曲线 |
| 天然 / 成分 | 原料实拍 / 成分图标 / 提取过程 |
| 保暖 / 透气 | 温度计 / 气流箭头 / 红外热图 |
| 高颜值 / 设计 | 几何线条 / 多角度展示 / 场景融入 |

> 卖点若无可视符号，visual-design 难以图文对应——文案定稿时即标注建议符号，避免「文图脱节」。

### 平台口吻差异化（按 `target_platform`）

同一组卖点，文案口吻按平台调性调整（受众与决策动机不同）：

| 平台 | 口吻 | 偏好 |
|---|---|---|
| **淘宝天猫** | 功能参数 + 价格敏感 | 参数/规格前置、利益直接、促销钩子明确 |
| **抖音** | 情绪冲击 + 短句节奏 | 痛点/反差前置、口语化短句、强代入场景 |
| **小红书** | 生活方式 + 真实感 | 体验/感受叙事、弱硬广、素人口吻 |
| **京东** | 参数 + 资质 + 服务 | 参数详尽、认证/服务标识、品质信任 |
| **微信小店** | 信任背书 + 私域 | 熟人推荐口吻、复购信任、社交亲密感 |

> 平台口吻影响主图文案措辞与详情页叙事节奏，但**核心卖点与 FABE 不变**，只换表达。

## 步骤 2：主图文案（5 张结构，CTR 优先）

主图 5 张是电商转化的第一战场。每张承担不同决策作用：

| 张 | 角色 | 文案要点 |
|----|------|----------|
| ① 点击主图 | CTR 之王，决定曝光转化 | 第一卖点 + 场景 + 价格/促销钩子；3 秒法则；信息层级：主体最大→卖点次之→钩子最小但醒目 |
| ② 细节/材质特写 | 建立品质信任 | 材质/工艺/细节卖点；微距质感 |
| ③ 使用场景 | 代入感与适用性 | 场景化利益（谁/何时/何处用）|
| ④ 对比/第二卖点 | 强化选择理由 | 与痛点/旧方案对比，或第二核心卖点 |
| ⑤ 品牌/资质/服务承诺 | 闭环信任 | 品牌背书、检测认证、售后（7天无理由/运费险/质保）|

**CTR 文案准则**：
- 数字前置、痛点前置、利益前置（先讲买家得到什么，再讲产品是什么）。
- 强对比（前后/有无/竞品对比）。
- 留白与层级：一张主图最多 1 主文案 + 1-2 辅文案，宁少勿挤。
- 钩子类（限时/赠品/包邮）须真实且合规（避免极限词）。

**产出**：`## 主图文案` 节，每张含：`张号 / 角色 / 主文案（≤12 字，图内大字）/ 辅文案（≤15 字）/ 必须出现的卖点关键词 / 禁用元素`。

## 步骤 3：详情页文案（黄金详情页结构 + FABE 叙事）

详情页是移动端竖屏长图叙事，按**黄金详情页结构**编排章节，每章对应一张 detail 图的文案：

1. **钩子首图**——承接主图，强化第一卖点/使用结果，留住下滑。
2. **痛点共鸣**——买家未被满足的痛点，制造需求。
3. **核心卖点解析**——第一卖点的 FABE 展开 + 可视化证据。
4. **场景化体验**——多场景使用利益，扩买家覆盖。
5. **对比证明**——与旧方案/竞品对比（有据，不贬损）。
6. **使用教程**——降低使用门槛与退货（怎么用/注意事项）。
7. **规格参数**——尺寸/材质/成分/适配，准确详尽（退货防线）。
8. **服务保障**——售后/物流/资质，闭环信任。
9. **品牌背书**（可选）——品牌故事/获奖/口碑。

**详情页文案准则**：
- 每节标题即卖点（买家快速扫读），正文用短句/列表/数字。
- 卖点图文一一对应：文案里提到的卖点，对应图里必须可视。
- 规格参数必须与产品档案一致（降低退货）。
- 章节数量按 `selected_modules` 的详情节数配置（默认 8-12）。

**产出**：`## 详情页文案` 节，每节含：`节号 / 章节角色 / 标题（≤12 字）/ 信息点列表 / 对应卖点 / 必须出现的文字`。

## 步骤 4：分享文案（站外引流，仅当已选 share_image）

短文案 + 社交钩子，**按分享渠道调口吻**（与 visual-design 的分享图口吻一致），含合规引导：

| 渠道 | 钩子风格 | 示例口吻 |
|---|---|---|
| **小红书** | 疑问/体验式、真实种草 | 「被我种草了…」「终于找到不踩雷的…」 |
| **微信（朋友圈/社群）** | 熟人推荐、复购信任 | 「自用回购 N 次」「私藏好物分享」 |
| **抖音** | 反差/利益前置、强冲击 | 「别再买错了」「用过就回不去」 |
| **淘宝/通用** | 对比/优惠钩子 | 「同款它更划算」「限时到手价」 |

**合规**：微信禁诱导分享/裂变（「转发得奖」「帮砍」违规）；小红书软广须标、禁明显导流；全渠道无极限词。

**产出**：`## 分享文案` 节，按渠道分条。

## 步骤 4.4：SKU 变体文案（仅当已选 sku_images）

为每个变体写**色名/规格标签**（用于 sku 图内文字与命名）：

- **色名**：用产品档案/原图色作准（如「雾霾蓝」「燕麦米」），不得臆造色卡外名称；色彩变体与原图色一致。
- **规格标签**：简洁准确（如「XL」「500ml」「清爽型」）。
- **变体差异化文案**（可选）：每变体 1 句场景化短句（如不同色的搭配建议），保持同系列统一。

**产出**：`## SKU 变体文案` 节。

---

## 步骤 4.5：去 AI 味（humanizer skill）

各模块文案（主图/详情页/分享）定稿后，**using the `humanizer` skill** 对全部文案做去 AI 改写，再进入合规扫描。电商文案重点针对：

- 广告式夸张与 rule-of-three（"赋能 / 打造 / 彰显 / 引领 / 开启全新"等空洞大词）
- AI 高频书面词与"不仅…而且"否定排比
- em dash（——）与 curly quotes（""''）滥用
- 空洞升华收尾（"让生活更美好"、"成就更好的自己"）

**改写而非删除**——保留每个卖点的 FABE 信息点、数字/对比/证据、价格钩子与转化逻辑（这是商业转化文案，每句话都要推动下单）。这是自动流水线步骤：不得调用 `AskUserQuestion`；没有写作样本时按产品档案、目标平台和当前文案语气直接改写。本步骤不调用任何 MCP 工具、不计费、无强度档位。

> ⚠️ **合规红线**：去 AI 改写**不得**为追求"人味/口语化"而引入《广告法》极限词（最/第一/极致/顶级/绝对/永久……）或无法证明的功效承诺。去 AI 完成后，仍须由 `ecommerce-platform-specs` 执行极限词扫描——两者顺序固定：**先去 AI，后合规**。

AI 写作模式、误报边界和改写方法以`humanizer` skill 为准；本节只定义电商业务约束。

---

## 合规前置（写文案时即规避，详细扫描由 ecommerce-platform-specs skill 执行）

- **《广告法》极限词禁用**：最/第一/国家级/顶级/极品/100%/绝对/永久/根治……（完整清单与平台违禁词由 `ecommerce-platform-specs` 提供，本 skill 写作时主动规避）。
- **不承诺无法证明的功效**（尤其美妆/健康/电子类）。
- **对比有据**，不贬损具体竞品。
